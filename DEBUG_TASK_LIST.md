# 任务列表加载失败排查

## 问题描述
用户反馈任务创建成功，但任务列表为空。这通常意味着：
1. `list_tasks` 接口在读取某个任务的 `config.json` 时抛出了异常。
2. 由于之前没有打印具体异常，异常被 `except` 块静默捕获并跳过，导致该任务消失。

## 修复内容
在 `backend/services/sign_tasks.py` 的 `list_tasks` 方法中添加详细的异常堆栈打印。

## 排查步骤
请同步代码并重新部署后，执行以下命令查看日志：
```bash
docker logs tg-signer-test-container
```
(如果在VPS上，请使用对应的容器名)

查找类似 `加载任务 xxx 失败:` 的日志输出，这将告诉我们具体是哪里出了问题（例如 JSON 解析错误、字段缺失等）。

## 其他
关于刷新页面返回首页，这通常是因为 Token 丢失或过期。请确认是否在同一浏览器会话中操作，并检查 LocalStorage。
